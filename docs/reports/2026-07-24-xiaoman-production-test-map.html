当前能测的生产路径

1

仓库内总校验

先确认 repo-side 实现和契约完整，再决定是否进入真实生产证据捕获。

2

Huabaosi 生产 Canary

验证首条真实生产图片 canary，产出可审计的 sanitized canary evidence。

3

QiWe 生产 Follow-up Deploy

如果 release/current 仍是普通 Huabaosi artifact，就切到独立 qiwe-production artifact。

4

真实小满活动证据

走完整条小满到企微生产链路，导出 real-activity evidence。

5

最终 Completion Manifest

把 staging 与 production 证据统一绑定到同一 released commit，完成最终校验。

一句话判断

现在不是继续补主流程代码 当前主要是 owner-operated 的生产执行与证据留档。
可以测到真实生产边界 Huabaosi canary、QiWe production follow-up、真实活动证据、群到达确认、final manifest 都有明确入口。
不能直接宣称 production-complete 只要真实 canary、真实 activity、群到达确认、completion manifest 还有一项没留证，就不能算完成。

现在怎么决定先测哪一步

Decision 01

本地总校验是否全绿

先跑 node tools/deploy/check-xiaoman-production-evidence-chain-local.mjs。

不绿：先回到仓库修正；全绿：进入真实 production evidence。
Decision 02

release/current 是否还是普通 Huabaosi artifact

如果还是普通 artifact，真实 activity 之前要补 QiWe production follow-up deploy。

仍是普通 artifact：先切 qiwe-production；已切：直接进真实 activity。
Decision 03

前置证据文件是否都已保留

Huabaosi canary、real activity、企微群到达确认、staging evidence 都要齐。

齐全：跑 finalizer；缺任意一项：不能宣称 production-complete。

按角色看现在该做什么

Owner / 发布人

决定是否进入真实生产证据捕获

确认 release 已发布，确认这次窗口就是要跑 Huabaosi canary、真实小满活动和最终 evidence retention。

执行人 / 刘珊

按 runbook 执行，不再扩主流程代码

依次跑 canary、必要时补 follow-up deploy、导出 real-activity evidence、保留企微群到达确认，最后跑 finalizer。

审阅人 / 交接人

只收 sanitized evidence

检查 retained 文件、artifact profile、两个 production sidecar SHA-256 是否分离，不接收 secrets 或 raw logs。
